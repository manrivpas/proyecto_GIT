Ramas o Branch

Son las líneas de tiempo con las instáneas creadas en cada momento
La principal por defecto recibe el nombre de master
Adicionalmente git permite crear ramas paralelas a la master a partir de una instantánea "commit"
Con esto creamos líneas de trabajo paralelas para pruebas o trabajos específicos del proyecto (p.e. una persona que solo trabaja en el diseño con css)
Podemos trabajar simultaneamente en la master y en las paralelas
El comando para crear una rama es: git branch nombre_de_la_rama
Por ejemplo: git branch javascript
Para ver en la rama que me encuentro ponemos en el terminal: git branch
Para trabajar en otra rama usamos: git checkout nombre_de_la_rama
Si hiciesemos checkout ahora para la rama master no se verían los cambios hechos en la otra rama
para fusionnar los cambios en las dos ramas se hace desde la master con: git merge nombre_de_la_rama_a_fusionar
